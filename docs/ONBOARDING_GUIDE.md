# Onboarding Guide

Welcome! Follow these steps to get started as a contributor:

## 1. Setup
- Clone the repo
- Copy `.env.example` to `.env.local` and fill in required secrets
- Run `npm install`
- Run `npm run build` to verify setup

## 2. Explore the Codebase
- Read `docs/PROJECT_REFERENCE_SUMMARY.md` (canonical source for patterns, architecture, and workflow)
- Review `docs/GOTCHAS_AND_FAQ.md` for common pitfalls
- See `docs/examples/` for sample data and flows

## 3. Running Locally
- Start the dev server: `npm run dev`
- Visit http://localhost:3000
- Test with example data if needed

## 4. Making Changes
- Follow the patterns in `PROJECT_REFERENCE_SUMMARY.md`
- Update docs if you add or change a workflow, integration, or convention
- Reference the doc update in your PR

## 5. Getting Help
- Ask in the team chat or open an issue
- See `docs/GOTCHAS_AND_FAQ.md` for troubleshooting
